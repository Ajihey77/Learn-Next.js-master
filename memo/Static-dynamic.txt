○ (Static) → 정적인 페이지로 미리 렌더링되어 배포됨 (빌드 시 HTML 생성)
ƒ (Dynamic) → 동적인 페이지로 요청 시 서버에서 렌더링됨 (요청 시 서버에서 렌더링됨)

모든 사용자에게 동일한 내용이 제공되는 페이지면: Static
사용자에 따라 다른 내용이 제공될 수 있는 페이지면: Dynamic

- 정적인 페이지를 동적으로 바꾸기
    - export const dynamic = "force-dynamic";
    ex )
        사용자 인증 정보 (매 요청마다 최신 상태 확인)
        실시간 변동 데이터 (주식 가격, 스포츠 경기 점수)
        관리 페이지에서 항상 최신 데이터를 보여줘야 할 때

- 정적인 페이지를 새로고침 시키기
    - export const revalidate = 60;
    ex )
        뉴스 기사 (1~5분마다 갱신)
        날씨 정보 (10~30분마다 갱신)
        인기 게시글 목록 (몇 분마다 업데이트)

- 동적인페이지로 만들고 캐시 이용하기
    1.  실시간 주식 가격 + 뉴스 목록 (캐시 + 동적 데이터)
        주식 가격: 항상 최신 데이터를 받아오기 위해 동적 데이터 처리.
        뉴스 목록: 5분마다 갱신되는 뉴스 목록은 캐시로 처리.

    2. 실시간 날씨 + 인기 상품 목록 (캐시 + 동적 데이터)
        날씨: 실시간으로 변하므로 매번 데이터를 새로 불러옴.
        인기 상품 목록: 10분마다 갱신되는 상품 목록을 캐시하여 사용.

    3. 실시간 스포츠 점수 + 사용자 리뷰 목록 (캐시 + 동적 데이터)
        스포츠 점수: 실시간으로 갱신되는 점수를 동적으로 가져오기.
        사용자 리뷰 목록: 1분마다 갱신되는 리뷰는 캐시하여 사용.
